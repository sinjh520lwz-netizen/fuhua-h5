+## 1. 回测引擎核心原理
+
+### 1.1 什么是回测
+
+回测（Backtesting）是用历史数据模拟交易策略的过程。核心问题只有一个：
+
+**如果过去用这个策略交易，会赚多少钱？**
+
+但这个问题远比看起来复杂。一个"看起来很赚钱"的回测，实盘可能亏得一塌糊涂。原因包括：未来函数、生存偏差、交易成本、滑点、流动性约束等。
+
+### 1.2 两种回测范式
+
+#### 范式一：向量化回测（Vectorized Backtesting）
+
+**原理**：把所有股票的所有数据放在一个大DataFrame里，用pandas/numpy的向量化操作一次性计算出结果。
+
+**适用场景**：
+- 策略逻辑简单（均线交叉、动量排序等）
+- 需要快速遍历大量参数组合
+- 研究阶段的因子筛选和策略初筛
+- 不需要模拟订单簿和成交细节
+
+**优点**：
+- 速度快（比事件驱动快100-1000倍）
+- 代码简洁
+- 容易并行化
+
+**缺点**：
+- 无法模拟复杂订单逻辑
+- 难以精确处理滑点和部分成交
+- 容易写出包含未来函数的代码
+
+#### 范式二：事件驱动回测（Event-Driven Backtesting）
+
+**原理**：模拟真实交易的时间流，按时间顺序逐根K线推送数据，策略收到数据后做出决策。
+
+**适用场景**：
+- 策略逻辑复杂（多品种对冲、条件单、追踪止损等）
+- 需要精确模拟成交过程
+- 需要接入实盘或模拟盘
+- 高频/日内策略
+
+**优点**：
+- 接近实盘环境

+- 代码结构清晰，容易迁移到实盘

+- 速度慢

+## 1. 过拟合的本质与检测
+
+### 1.1 什么是过拟合
+
+过拟合（Overfitting）是量化交易中最大的敌人。简单来说：
+
+**你的策略在历史数据上表现很好，但在新数据上表现很差。**
+
+就像一个学生把所有练习题的答案都背下来了，考试成绩很好，但遇到新题就不会做。
+
+过拟合的根本原因：
+1. **参数太多，数据太少**：策略有10个参数，但只有50次交易，很容易"记住"噪音
+2. **过度优化**：反复在同一段数据上调整参数，直到找到"完美"的参数
+3. **数据窥探（Data Snooping）**：看了太多次数据，无意中选择了恰好有效的模式
+4. **选择偏差**：测试了100个策略，只报告最好的那个
+
