**数据流向说明**:

| 流向 | 描述 | 同步方式 |
|------|------|----------|
| 移动点餐 → 收银端 | 服务员下单后实时同步到收银台 | WebSocket 推送 |
| 收银端 → KDS | 订单提交后推送至厨房大屏 | WebSocket + 打印 |
| 海鲜池 → 收银端 | 称重数据实时同步，自动计算金额 | WebSocket 推送 |
| KDS → 收银端 | 出餐状态回传、估清信息同步 | WebSocket 推送 |
| 收银端 → 打印机 | 按档口分单打印小票 | HTTP 调用打印服务 |

### 2.2 核心表结构

#### 2.2.1 桌台表 `tables`

```sql
CREATE TABLE `tables` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `table_no` VARCHAR(10) NOT NULL COMMENT '桌号，如 A01, B12',
  `area` VARCHAR(20) NOT NULL COMMENT '区域：hall(大厅)/private(包厢)/outdoor(露台)',
  `floor` TINYINT NOT NULL DEFAULT 1 COMMENT '楼层',
  `seats` TINYINT NOT NULL DEFAULT 4 COMMENT '座位数',
  `status` ENUM('free','occupied','reserved','cleaning') NOT NULL DEFAULT 'free',
  `current_order_id` BIGINT UNSIGNED DEFAULT NULL COMMENT '当前订单ID',
  `qr_code` VARCHAR(255) DEFAULT NULL COMMENT '桌台二维码URL',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_table_no` (`table_no`),
  KEY `idx_status` (`status`),
  KEY `idx_area` (`area`, `floor`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='桌台表';
```

## 三、API接口清单

### 3.1 接口规范

- **Base URL**: `http://api.example.com/v1`
- **认证**: JWT Token，放在 `Authorization: Bearer <token>` 头
- **响应格式**: `{ code: 0, message: "success", data: {} }`
- **错误码**: `0`=成功, `1001`=参数错误, `1002`=未授权, `1003`=无权限, `2001`=订单不存在, `2002`=菜品已估清, `2003`=版本冲突

### 3.2 接口清单

#### 3.2.1 认证模块 `/auth`

| 方法 | 路径 | 描述 | 权限 |
|------|------|------|------|
| POST | `/auth/login` | 员工登录 | 公开 |
| POST | `/auth/logout` | 退出登录 | 登录 |
| GET | `/auth/profile` | 获取当前用户信息 | 登录 |
| PUT | `/auth/password` | 修改密码 | 登录 |

#### 3.2.2 桌台模块 `/tables`

| 方法 | 路径 | 描述 | 权限 |
|------|------|------|------|
| GET | `/tables` | 获取桌台列表（支持区域筛选） | 登录 |
| GET | `/tables/:id` | 获取桌台详情 | 登录 |
| PUT | `/tables/:id/status` | 更新桌台状态 | 收银/服务员 |
| POST | `/tables/:id/open` | 开台（关联订单） | 收银/服务员 |
| POST | `/tables/:id/clear` | 清台 | 收银 |

#### 3.2.3 菜品模块 `/dishes`

| 方法 | 路径 | 描述 | 权限 |
|------|------|------|------|
| GET | `/dishes` | 获取菜品列表（支持分类筛选） | 登录 |
| GET | `/dishes/:id` | 获取菜品详情 | 登录 |
| GET | `/dishes/categories` | 获取菜品分类列表 | 登录 |

## 四、WebSocket事件定义及消息格式

### 4.1 连接与认证

```javascript
// 客户端连接
const socket = io('ws://api.example.com', {
  auth: { token: 'jwt_token_here' },
  reconnection: true,
  reconnectionDelay: 1000,
  reconnectionAttempts: 10
});

// 认证成功后加入房间
socket.on('connected', (data) => {
  // data.role: 'cashier' | 'kitchen' | 'seafood' | 'waiter'
  // 根据角色自动加入对应房间
  socket.emit('join', { rooms: getRoomsByRole(data.role) });
});
```

### 4.2 事件定义

#### 4.2.1 订单相关事件

| 事件名 | 方向 | 触发场景 | 消息格式 |
|--------|------|----------|----------|
| `order:created` | Server→Client | 新订单创建 | `{ orderId, orderNo, tableNo, items[], createdBy }` |
| `order:updated` | Server→Client | 订单变更（加菜） | `{ orderId, orderNo, addedItems[], totalAmount, version }` |
| `order:item-cancelled` | Server→Client | 菜品取消 | `{ orderId, itemId, reason }` |
| `order:item-returned` | Server→Client | 退菜 | `{ orderId, itemId, reason, returnedBy }` |
| `order:settled` | Server→Client | 订单结账 | `{ orderId, tableNo, payAmount, payMethod }` |
| `order:cancelled` | Server→Client | 订单取消 | `{ orderId, tableNo, reason }` |

#### 4.2.2 KDS相关事件

| 事件名 | 方向 | 触发场景 | 消息格式 |
|--------|------|----------|----------|
| `kds:new-item` | Server→Client | 新菜品推送至KDS | `{ itemId, dishName, quantity, stationId, tableNo, cookRemark, isRush, createdAt }` |
| `kds:item-status` | Server→Client | 菜品状态变更 | `{ itemId, status, stationId }` |
| `kds:rush` | Server→Client | 催菜提醒 | `{ itemId, dishName, tableNo, rushTime }` |
| `kds:item-estimated` | Server→Client | 菜品估清 | `{ dishId, dishName }` |

#### 4.2.3 海鲜池相关事件

| 事件名 | 方向 | 触发场景 | 消息格式 |
|--------|------|----------|----------|
| `seafood:weigh-request` | Server→Client | 待称重请求 | `{ itemId, dishName, tableNo, estimatedWeight }` |
| `seafood:weigh-done` | Server→Client | 称重完成 | `{ itemId, actualWeight, amount, photoUrl }` |
| `seafood:weight-sync` | Server→Client | 重量同步至收银 | `{ orderId, itemId, weight, amount }` |

#### 4.2.4 桌台相关事件

| 事件名 | 方向 | 触发场景 | 消息格式 |
|--------|------|----------|----------|

#### 4.2.5 系统事件

| 事件名 | 方向 | 触发场景 | 消息格式 |
|--------|------|----------|----------|
| `system:broadcast` | Server→Client | 系统广播 | `{ type, message }` |
| `system:sync-request` | Server→Client | 同步请求（断线恢复） | `{ lastSyncAt }` |
| `system:heartbeat` | Bidirectional | 心跳检测 | `{ timestamp }` |

### 4.3 消息推送路由（房间机制）

```javascript
// 房间划分
const ROOMS = {
  CASHIER_MAIN: 'cashier:main',      // 主收银机
  CASHIER_SUB: 'cashier:sub',        // 副收银机
  KDS_ALL: 'kds:all',               // 所有KDS屏幕
  KDS_STIR_FRY: 'kds:stir_fry',     // 炒菜档KDS
  KDS_SASHIMI: 'kds:sashimi',       // 刺身档KDS
  KDS_STEAMED: 'kds:steamed',       // 海鲜蒸煮档KDS
  KDS_COLD: 'kds:cold',             // 凉菜档KDS
  SEAFOOD: 'seafood:pool',          // 海鲜池
  WAITER: 'waiter:all',             // 服务员
  ADMIN: 'admin'                     // 管理员
};

// 事件推送路由
const ROUTES = {
  'order:created':    [CASHIER_MAIN, CASHIER_SUB, KDS_ALL, WAITER],
  'kds:new-item':     [KDS_STIR_FRY, KDS_SASHIMI, KDS_STEAMED, KDS_COLD], // 按档口路由
  'seafood:weight-sync': [CASHIER_MAIN, CASHIER_SUB],
  'table:status-changed': [CASHIER_MAIN, CASHIER_SUB, WAITER],
  'dish:estimated':   [CASHIER_MAIN, CASHIER_SUB, KDS_ALL, WAITER, SEAFOOD]
};
```

### 4.4 断线重连与增量同步

```javascript
// 客户端断线重连逻辑
socket.on('reconnect', async () => {
  const lastSyncAt = localStorage.getItem('lastSyncAt');
  
  // 请求增量同步
  socket.emit('sync:request', { 
    lastSyncAt,
    clientId: getClientId(),
    role: getClientRole()
  });
});

// 服务端增量同步响应
socket.on('sync:response', (data) => {
  // data.orders: 增量变更的订单
  // data.tables: 桌台状态变更
  // data.dishes: 菜品估清变更
  applyIncrementalSync(data);
  localStorage.setItem('lastSyncAt', data.syncAt);
});
```

```typescript
// 后端：更新订单时检查版本号
async updateOrder(orderId: number, data: any, clientVersion: number) {
  const result = await prisma.orders.updateMany({
    where: {
      id: orderId,
      version: clientVersion  // 版本号匹配才更新
    },
    data: {
      ...data,
      version: { increment: 1 }  // 版本号+1
    }
  });
  
  if (result.count === 0) {
    // 版本冲突，返回最新数据让客户端合并
    const latest = await prisma.orders.findUnique({ where: { id: orderId } });
    throw new ConflictError('数据已被其他人修改，请刷新后重试', latest);
  }
}
```

```typescript
// ESC/POS 打印指令封装
class ESCPOSBuilder {
  buffer: Buffer[] = [];
  
  // 初始化
  initialize() {
    this.buffer.push(Buffer.from([0x1B, 0x40]));
    return this;
  }
  
  // 居中对齐
  alignCenter() {
    this.buffer.push(Buffer.from([0x1B, 0x61, 0x01]));
    return this;
  }
  
  // 加粗
  bold(on: boolean) {
    this.buffer.push(Buffer.from([0x1B, 0x45, on ? 0x01 : 0x00]));
    return this;
  }
  
  // 字体放大
  doubleSize() {
    this.buffer.push(Buffer.from([0x1D, 0x21, 0x11]));
    return this;
  }
  
  // 打印文字
  text(str: string) {
    this.buffer.push(Buffer.from(str, 'utf8'));
    return this;
  }
  
  // 换行
  newLine(count = 1) {
    for (let i = 0; i < count; i++) {
      this.buffer.push(Buffer.from([0x0A]));
    }
    return this;
  }
  
  // 切纸
  cut() {
    this.buffer.push(Buffer.from([0x1D, 0x56, 0x00]));
    return this;
  }
  
  // 构建打印内容
  build() {
    return Buffer.concat(this.buffer);
  }
}

// 打印厨房小票
function printKitchenTicket(item: OrderItem, tableNo: string) {
  const builder = new ESCPOSBuilder();
  
  return builder
    .initialize()
    .alignCenter()
    .bold(true)
    .doubleSize()
    .text(`【${item.stationName}】`)
    .newLine()
    .bold(false)
    .text(`桌号: ${tableNo}`)
    .newLine()
    .text(`菜品: ${item.dishName} x${item.quantity}`)
    .newLine()
    .text(`做法: ${item.cookingRemark || '无'}`)
    .newLine()
    .text(`时间: ${formatTime(item.createdAt)}`)
    .newLine(2)
    .text(item.isRush ? '★ 催菜 ★' : '')
    .newLine(2)
    .cut()
    .build();
}
```

### 8.3 离线同步策略

```typescript
// IndexedDB 离线存储
class OfflineStorage {
  private db: IDBDatabase;
  
  async init() {
    return new Promise((resolve, reject) => {
      const request = indexedDB.open('restaurant_offline', 1);
      
      request.onupgradeneeded = (event) => {
        const db = (event.target as IDBOpenDBRequest).result;
        
        // 离线订单存储
        if (!db.objectStoreNames.contains('pending_orders')) {
          const store = db.createObjectStore('pending_orders', { keyPath: 'localId' });
          store.createIndex('tableNo', 'tableNo');
          store.createIndex('createdAt', 'createdAt');
        }
        
        // 同步队列
        if (!db.objectStoreNames.contains('sync_queue')) {
          const store = db.createObjectStore('sync_queue', { keyPath: 'id', autoIncrement: true });
          store.createIndex('type', 'type');
          store.createIndex('timestamp', 'timestamp');
        }
      };
      
      request.onsuccess = (event) => {
        this.db = (event.target as IDBOpenDBRequest).result;
        resolve();
      };
    });
  }
  
  // 保存离线订单
  async savePendingOrder(order: any) {
    const tx = this.db.transaction('pending_orders', 'readwrite');
    const store = tx.objectStore('pending_orders');
    order.localId = `local_${Date.now()}`;
    order.synced = false;
    await store.add(order);
    return order.localId;
  }
  
  // 获取待同步队列
  async getSyncQueue() {
    return new Promise((resolve) => {
      const tx = this.db.transaction('sync_queue', 'readonly');
      const store = tx.objectStore('sync_queue');
      const request = store.getAll();
      request.onsuccess = () => resolve(request.result);
    });
  }
  
  // 网络恢复后同步
  async syncPendingOrders() {
    const pending = await this.getPendingOrders();
    
    for (const order of pending) {
      try {
        // 提交到服务器
        const result = await api.createOrder(order);
        
        // 更新本地状态
        order.synced = true;
        order.serverId = result.id;
        await this.updatePendingOrder(order);
        
        // 从队列移除
        await this.removeFromQueue(order.localId);
      } catch (error) {
        // 同步失败，保留在队列
        console.error('Sync failed:', error);
      }
    }
  }
}
```
